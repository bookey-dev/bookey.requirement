## 规范

### 需求 `issues` 命名

- `Feat - [需求]` - 功能
- `Fix - [需求]` - 修复、优化
- `Docs - [需求]` - 文案、文档
- `Style - [需求]` - 样式、UI
- `Refactor - [需求]` - 重构、变更
- `Test - [需求]` - 测试
- `Chore - [需求]` - 其他、删除
